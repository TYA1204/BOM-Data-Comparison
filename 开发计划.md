# BOM Data Comparison 工具开发计划

> 基于全网调研与制造业最佳实践制定 | 2025-06-03
> **约束：SAP API不可用，数据源仅限本地文件导入（Excel/CSV）**

---

## 一、项目概述

### 1.1 业务痛点

| 痛点 | 现状描述 | 影响 |
|------|----------|------|
| **人工比对效率低** | 整机BOM物料品类繁杂，各结构BOM同类物料序号、字段规范标准不一，全依赖人工目视比对 | 长时间工作引发视觉疲劳，差错率高 |
| **效率与耗时** | 现行对标作业方式效率低下、耗时较长 | 占用大量人力成本 |
| **追溯困难** | 人工核对时少校准/漏存记录，核查流程与问题整理难以追溯 | 无法形成知识沉淀 |
| **隐匿问题难排查** | 多批次换产带来不眠物料，物料差异化无法提前预警排查 | 易引起产成品虚工、物料呆滞等问题，拉高生产成本 |

### 1.2 项目目标

引入AI技术开发 **BOM差核对比工具**，替换传统人工核验模式，实现：

- 自动识别物料差异（错漏、规格、用量、位置）
- 高效率准确比对，规避人工核查短板
- 提升物料核对工作的规范性与准确度

### 1.3 预期效果

| 指标 | 目标值 |
|------|--------|
| 人工核对时间缩短 | **≥80%** |
| 差异识别准确率 | **≥99%** |
| 支持比对模式 | 新旧版本、跨机型BOM一键比对 |
| 差异标注 | 自动识别并高亮标注差异项 |
| 输出产物 | 结构化差异物料清单（Excel/PDF） |
| 追溯能力 | 核对过程全记录，差异数据可沉淀 |

### 3.2 差异比对规则引擎

#### 3.2.1 物料匹配规则

```python
匹配策略：
├── 精确匹配 → 物料编号完全一致
│   └── 优先级最高
├── 模糊匹配 → rapidfuzz计算相似度
│   ├── 相似度 ≥ 95% → 视为同一物料
│   ├── 相似度 80%-95% → 标记为"疑似匹配"，需人工确认
│   └── 相似度 < 80% → 视为不同物料
└── 别名映射 → 维护物料别名对照表
    └── 解决不同系统间编码体系差异
```

#### 3.2.2 差异分类规则

| 差异类型 | 判定条件 | 严重程度 |
|----------|----------|----------|
| **新增物料** | 仅存在于BOM B，BOM A中无匹配 | 🔴 高 |
| **移除物料** | 仅存在于BOM A，BOM B中无匹配 | 🔴 高 |
| **数量变更** | 同一物料，用量不一致 | 🟡 中 |
| **位号变更** | 同一位号对应不同物料 | 🔴 高 |
| **规格变更** | 同一物料，规格参数不一致 | 🟡 中 |
| **版本变更** | 同一物料，版本号不一致 | 🟢 低 |
| **替代料变更** | 替代料关系增删改 | 🟡 中 |
| **层级变更** | BOM结构树节点位置变化 | 🟡 中 |

#### 3.2.3 位号变更细分规则（参考BOMCompare最佳实践）

```
位号变更分类：
├── [常规替换]
│   └── 同一位号，BOM A物料X → BOM B物料Y，且Y在A中已存在
│       → 判定为"已有物料的重新分配"
├── [新物料替换]
│   └── 同一位号，BOM A物料X → BOM B物料Y，且Y在A中不存在
│       → 判定为"引入全新物料"
└── [物料整合]
    └── 多个位号在A中对应不同物料，在B中合并为同一物料
        → 判定为"N:1物料合并"
```

### 3.3 数据模型设计

#### 3.3.1 BOM表结构

```sql
-- BOM主表
CREATE TABLE bom_header (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    bom_name VARCHAR(255) NOT NULL,           -- BOM名称/型号
    bom_version VARCHAR(50),                   -- BOM版本号
    source_type VARCHAR(50),                   -- 来源：Excel/CSV/Manual
    source_file VARCHAR(500),                  -- 原始文件名
    total_items INTEGER,                       -- 总物料数
    total_quantity INTEGER,                    -- 总用量
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- BOM明细表
CREATE TABLE bom_item (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    bom_id INTEGER NOT NULL,
    line_no INTEGER,                           -- 行号
    level INTEGER,                             -- BOM层级
    parent_pn VARCHAR(100),                    -- 父件物料号
    part_number VARCHAR(100) NOT NULL,         -- 物料编码
    part_name VARCHAR(500),                    -- 物料名称
    specification VARCHAR(1000),               -- 规格描述
    quantity DECIMAL(10,4),                    -- 用量
    unit VARCHAR(20),                          -- 单位
    reference_designator VARCHAR(500),         -- 位号/参考位置
    version VARCHAR(50),                       -- 版本
    manufacturer VARCHAR(200),                 -- 制造商
    mpn VARCHAR(200),                          -- 制造商料号
    alternative_parts TEXT,                    -- 替代料（JSON数组）
    FOREIGN KEY (bom_id) REFERENCES bom_header(id)
);

-- 比对任务表
CREATE TABLE comparison_task (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_name VARCHAR(255),
    source_bom_id INTEGER NOT NULL,            -- 基准BOM
    target_bom_id INTEGER NOT NULL,            -- 对比BOM
    status VARCHAR(50),                        -- pending/running/completed/failed
    comparison_type VARCHAR(50),               -- version_cross/ cross_model
    created_by VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP,
    FOREIGN KEY (source_bom_id) REFERENCES bom_header(id),
    FOREIGN KEY (target_bom_id) REFERENCES bom_header(id)
);

-- 差异结果表
CREATE TABLE comparison_result (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id INTEGER NOT NULL,
    diff_type VARCHAR(50) NOT NULL,            -- added/removed/modified
    diff_category VARCHAR(50),                 -- material/quantity/reference/spec/version
    severity VARCHAR(20),                      -- high/medium/low
    part_number_a VARCHAR(100),                -- BOM A物料号
    part_number_b VARCHAR(100),                -- BOM B物料号
    field_name VARCHAR(100),                   -- 差异字段
    old_value TEXT,                            -- 原值
    new_value TEXT,                            -- 新值
    reference_a TEXT,                          -- BOM A位号
    reference_b TEXT,                          -- BOM B位号
    match_confidence DECIMAL(5,2),             -- 匹配置信度
    is_confirmed BOOLEAN DEFAULT FALSE,        -- 是否已确认
    confirmed_by VARCHAR(100),
    confirmed_at TIMESTAMP,
    FOREIGN KEY (task_id) REFERENCES comparison_task(id)
);
```
