# BOM 核对差异 — 保姆级操作指南

> 打开浏览器，输入 `http://172.20.217.12:40045/compare/`，照着做就行。

---

## 第一步：传文件（把活儿接进来）

### 1.1 打开系统

浏览器里输入地址，回车。看到页面上方有「首页」、「上传BOM」、「开始比对」三个标签，点「上传BOM」。

### 1.2 上传 BOM 文件

- 点「选择文件」按钮，找到你手里那份 BOM 文件（.xls 或 .xlsx 都行）
- 文件选好后，点「上传解析」
- 等几秒，页面提示"上传成功"，右边会显示解析出来的 BOM 树
- 给这个 BOM 起个名字（比如"100P3EM 基准版"），方便后面认

### 1.3 上传第二份 BOM

回到上传页，同样操作把第二份也传上去。起个区分的名字（比如"100H5F 新版"）。

**注意**：两份都上传完才能去比对，少一份比不了。

---

## 第二步：选模式（想怎么比）

### 2.1 切到比对页面

点顶部的「比对」标签，进入比对主界面。

### 2.2 选种子（定范围）

页面左边会显示两份 BOM 的组件树，最上面是几个大类：

```
电子BOM、大配管BOM、结构BOM、专利组件
```

- **全比**：默认所有组件都勾选上了，别动它就行
- **只比某几个**：把不想比的复选框点掉

### 2.3 选比对类型

页面上方有一排按钮，看清楚选：

| 按钮           | 干啥用                                            |
| -------------- | ------------------------------------------------- |
| **版本比对**   | 同一机型，新版跟旧版比（比如 P3EM v2 vs P3EM v1） |
| **跨机型比对** | 不同机型互相对比（比如 100P3EM vs 100H5F）        |

点一下变蓝就是选中了。

### 2.4 选对比范围

下面有两个圆按钮（radio），只能选一个：

| 选项                     | 比什么                               | 什么时候用                                     |
| ------------------------ | ------------------------------------ | ---------------------------------------------- |
| **组件对比（排除物料）** | 只比组件（有子件的），叶子物料不参与 | 想看整体结构变化，忽略螺丝/贴纸/包装袋这些零碎 |
| **全层级对比（含物料）** | 所有东西都拿出来对比，组件+物料全上  | 想看到每个细节差异，大到面板小到贴纸全查       |

> 💡 建议：先用「组件对比」看大局，再用「全层级」看细节。

### 2.5 开跑

点「开始比对」按钮。

> 💡 不想比某个组件？把它的复选框点掉就行。不勾的组件及其全部子件会从比对中完全移除。

---

## 第三步：看结果（差异都在这了）

### 3.1 差异概览

比完后页面跳转到结果区，顶部有一行筛选按钮：

```
组件 14  |  物料 15  |  数量 1  |  单位 0  |  版本 0
```

数字代表该类差异有多少条。点某个按钮可以只看那一类。

### 3.2 差异明细解读

每条差异长这样：

| 字段         | 啥意思                                                |
| ------------ | ----------------------------------------------------- |
| **新增**     | 新BOM里有，旧BOM里没有 → 人家加料了                   |
| **删除**     | 旧BOM里有，新BOM里没有 → 人家减料了                   |
| **数量变更** | 同料号，数量变了 → 用量改了                           |
| **单位变更** | 同料号，单位变了（比如"个"变"套"）→ 包装/计量方式改了 |
| **版本变更** | 同料号，版本号变了 → 物料升级了（仅版本比对可见）     |

### 3.3 排序翻看

表头每个字段都能点，点一下按这个字段排，再点一下倒过来排。方便你按父组件分组查看。

### 3.4 搜索筛选

差异明细上方有个搜索框，输入料号或名称关键词，立刻定位到你要找的那一条。

---

## 第四步：出报告（别只盯着屏幕看）

### 4.1 填写导出配置

比对完成后，左侧会自动显示「导出配置」卡片。填好这五项：
- **阶段**：DVT / PVT / MP（下拉选）
- **数量**：填数字，如 1
- **订单号**：如 2606002KL
- **拟制**：你的名字
- **审核**：审核人名字

填一次就记住了，下次打开自动回填，不用重复填。

### 4.2 导出 WORD 更改通知单

点差异明细右上角蓝色「导出 Word 更改通知单」按钮。如果导出配置有漏填的会提示补上，补完再点。

WORD 报告自动按父组件分组，叶子差异占一行，蓝色粗体组件头 + 深灰色物料行，物料用量对齐。文件名格式：`{订单号}_{机芯}_{机型}_{数量}台_{阶段}_整机更改通知单.docx`

### 4.3 导出 Excel 报告

差异明细右上角「导出 Excel 报告」按钮，下下来是一份结构化表格，方便二次加工。

---

## 第五步：盯重点（别被数据淹了）

### 5.1 先看组件级变化

把筛选器里的「组件」点亮，先看结构性的东西——哪个组件整个换了、哪个组件新增了。这些是大事，先处理。

### 5.2 再看物料变化

把「物料」点亮，看叶子级的零碎变化——贴纸换了、说明书变了、遥控器型号升了一级。这些是小事，但对采购和仓库影响大。

### 5.3 特别留意数量变了但料号没变的

这种是"隐形变更"——料还是那个料，但一份BOM里用2个，另一份用3个。数量变了意味着成本变了，必须跟研发确认。

### 5.4 不同父组件的同料号不算变更

同一个料号如果分别挂在 P3EM 和 H5F 的不同父组件下面，系统不会瞎报"数量变更"。它会找同父组件下的同料号对比，差一个层级就不比。

---

## 第六步：存档收工

### 6.1 命名规范

WORD 导出自动使用规范命名，不用手动改：

```
2606002KL_8R713_100P3EM_2台_DVT_整机更改通知单.docx
{订单号}_{机芯}_{机型}_{数量}台_{阶段}_整机更改通知单.docx
```

### 6.2 拉人确认

报告拉出来，找设计师、物料、工艺相关的人一起过一遍：

- **研发**：确认新增/删除的组件是不是计划内的
- **物料**：数量变了要不要多备货

### 6.3 更新主 BOM

定下来的结果更新到正式的 BOM 里，更新人、更新日期、变更原因都写清楚。

### 6.4 保留历史

原始的两份 BOM 文件和导出的差异报告别删，打包存一份。下次有人问"上次比是什么结果"，你直接翻出来。

## 常见坑（踩过才长记性）

| 坑                             | 现象                                                | 解法                                               |
| ------------------------------ | --------------------------------------------------- | -------------------------------------------------- |
| **缓存没清**                   | 明明改了代码，页面还是老样子                        | 刷新页面（Ctrl+F5 强制刷新）                       |
| **选了单选还出全量**           | 你以为只比一个组件，结果全比了                      | 对比范围用「全层级对比（含物料）」                 |
| **两边数量一样但物料显示差异** | 单位不一样（个 vs 百个），数量数值一样但实际差100倍 | 看结果里的"单位变更"分类，这个系统会自动揪出来     |
| **料号一样，规格不同没报**     | 系统默认只比料号、数量、单位、版本，不比规格描述    | 规格差异要手动看，系统目前不做                     |
| **BOM 上传失败**               | 文件格式不对或数据太乱                              | 先把 BOM 原文件在 Excel 里打开另存为 .xlsx，再上传 |

---

## 一句话总结

**上传两份 BOM → 选对模式 → 点开始比对 → 看差异 → 导出报告 → 拉人确认 → 更新主 BOM → 存档收工。**
