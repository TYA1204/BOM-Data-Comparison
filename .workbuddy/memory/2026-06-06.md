## 修复位号(reference)数据丢失问题

### 根因
SAP BOM 展开表在大位号列表时，位号会**跨行续写**。续行特征：col[2] 为空，col[0] 为位号格式(如 C14S11)。原解析器 `extract_data_row()` 只读取当前行 col[25+] 的位号，丢失了所有续行位号。续行跨页断后仍然继续，未处理跨页续行场景。

### 修复内容
1. **`bom_cleaner.py`** 新增：
   - `REF_CONTINUATION_PATTERN` 正则（匹配 C/R/L/Q/D/FB 等位号前缀）
   - `_PAGE_KW_RE` 正则（过滤页眉/页脚关键词）
   - `_is_ref_continuation(cols)` 函数（判断是否为位号续行）
2. **主解析循环**：
   - `for line in lines:` → `while i < total_lines:` 索引循环，支持 look-ahead
   - 每个 data row 处理完后扫描续行：`lines[i].strip().split('\t')` 读取，跳过页眉页脚直到下一个 data row
   - 续行所有列中的位号合并到 data row 的 reference 字段
3. **关键修复点**：续行扫描必须使用 `lines[i].strip().split('\t')`（非 `split()`），因续行有前导 tab 导致 col[0] 为空
