## 「仅看差异」升级为方案B（拍平父组件清单）

**背景**：原「仅看差异」保留"叶子+爹+全部祖先"整条链路，用户觉得链路太长，只想看"哪个直接父组件下面有差异"。

**受限点**：组件树是嵌套 DOM（爷爷⊃爹⊃儿子），CSS 里爷爷 display:none 会连爹一起隐藏，"显示爹藏爷爷"在树形态里物理做不到。→ 只能脱离树，拍平成清单。

**确认方案（AskUserQuestion）**：
- 方案B 拍平父组件清单（替代原树 + 勾选爹导出整段）
- 开启「仅看差异」时该侧从嵌套树切换为平铺清单，关闭恢复完整树

**实现（全在 compare.html + app.css，纯前端）**：
- 全局变量：`diffParentList = {source:[], target:[]}`、`compNameMap`（pn→name，buildParentMap 顺带建）、保留 onlyDiffMode
- `buildDiffParentList()`：遍历 allResults，跳过 diff_category==='component'，按 parent_pn_a/parent_pn_b 聚合直接父组件，每个爹附差异叶子明细数组
- `renderDiffParentList(side)`：平铺表格（复选框+PN+名称+差异叶子数徽标），点击行展开看差异叶子明细
- `rebuildTreeView(side)`：onlyDiffMode 时调 renderDiffParentList，否则 renderComponentTree
- 触发时机：loadResult 拿到 allResults 后 + loadComponents 加载树后各构建一次（防名称为空）
- 勾选/导出零改动：导出走 getSelectedComponentPNs 读 tbody `.comp-{side}-cb` 勾选值，平铺清单复选框复用同 class，勾爹导出整段自动生效

**验证**：
- Node 模拟 6 场景全过（聚合/过滤/勾选导出）
- 所有内联 script JS 语法校验通过
- 部署：备份→scp 上传 compare.html+app.css→stop 清缓存→run.sh --force
- 服务器校验：gunicorn 5 worker 监听 40045、新函数 8 处引用、0 pycache 残留、HTTP 200
