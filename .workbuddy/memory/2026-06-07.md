### 5. 组件识别逻辑修复
- **问题**：SAP 导出的 BOM 数据 unit 字段全为空，原逻辑 `unit='ST'` 导致 "无组件" 显示
- **修复**：改为基于 BOM 层级结构识别组件 — `part_number` 出现在 `parent_pn` 中即为组件
- 涉及文件：
  - `app/routes/upload.py` — `get_components()` 和 `export_all_components()` 改用 `EXISTS` 子查询
  - `app/templates/upload.html` — 文本更新（"ST 单位" → "基于层级结构"）
- 修复后 959 条数据识别出 48 个组件

### 6. 主BOM/副BOM 类型选择修复
- **问题**：`bom_type` 列存在但插入不写、上传不传，全默认 primary
- **修复**：3 处联动改动
  - `upload.html` — 新增「BOM 类型」下拉框（主 BOM / 副 BOM）
  - `upload.py` — 接收 `bom_type` 参数传入 `parse_bom_file()`
  - `parser.py` — 函数签名加 `bom_type` 参数，两处 INSERT 加入该字段

### 7. 单位/位号字段混叠修复
- **根因**：SAP BOM 的 `extract_data_row` 硬编码 col[24]=unit, col[25+]=reference
  但不同 SAP 导出格式列布局不同，部分格式 unit 为空、混入 reference 首位
- **修复**：`bom_cleaner.py` — `extract_data_row` 增加 unit 为空时的智能检测
  若 reference 首位是已知单位（ST/PC/PCS等），自动分离

### 8. PC子件向上归并为ST父组件
- **需求**：导出Excel中PC子件物料名称替换为父级ST组件名称，导出结果只应出现ST
- **实现**：`reporter.py` — `generate_components_export_excel` 新增 `find_st_ancestor_name()`
  对每个非ST子件沿 `parent_pn` 链向上追溯最近ST祖先，替换 `part_name`
- **验证**：418个PC件全部替换为对应ST组件名称

### 9. 相同BOM对比产生大量虚假差异修复
- **根因**：BOM 中存在重复物料号，`differ.py` 用 `matched_items_b[0]` 取第一条匹配
  导致第 2、3...n 次出现的同料号全部与第一条比较，产生 95 个虚假差异
- **修复（3处）**：
  1. Section 3 字段对比 → 新增 `_pick_by_parent()` 按 (物料号+父件) 精确匹配
  2. Section 6 层级检测 → 同样用 `_pick_by_parent()` 选取
  3. Section 5 物料整合 → 双向检验 `source_pns != target_pns`
- **验证**：相同文件两次上传后对比，差异从 95 → 0，ST自身保持原名
