## 启动脚本彻底修复（第3次）

### 根因
双击 `.bat` 闪退的**真正根因**：`for` 循环内部 `echo` 语句中的 `...`（三个英文句点）在中文 Windows cmd.exe 中触发 `此时不应有 ...` 语法错误，导致脚本异常终止，窗口闪退。

### 修复内容（完全重写）
1. **替换 `...` 为 `--`** — `for` 循环内的 echo 不再使用三连点
2. **`setlocal enabledelayedexpansion`** — 启用延迟变量扩展，`!FOUND!`、`!READY!` 变量正常工作
3. **`chcp 65001`** — 强制 UTF-8 编码，避免中文字符乱码
4. **`ping` 替代 `timeout`** — `timeout` 非所有 Windows 版本可用；`ping 127.0.0.1 -n N` 通用兼容
5. **`goto` 移出 `for /L`** — 避免跳转破坏循环栈导致语法错误
6. **`start ... /MIN` 后台启动 Flask** — 不阻塞主脚本窗口，可同时显示状态和 `pause`
7. **`start "" http://localhost:5002`** — 自动打开浏览器（之前缺失）
8. **`startup.log` 日志** — 每步操作写入日志，出错可溯源
9. **Flask 导入预检** — Step 2 先验证应用可导入，避免启动后才发现依赖缺失
10. **服务器就绪等待** — 轮询端口最多10秒，超时则报错

### 测试验证
- ✅ Flask 应用导入正常
- ✅ 端口5002监听正常
- ✅ 页面 http://localhost:5002 可访问

## 修复「清除全部数据」自增ID未重置Bug

### Bug
点击「清除全部数据」后，`DELETE FROM` 清了4张表的数据，但 SQLite 内部 `sqlite_sequence` 表未被清理，
新插入数据 ID 从旧最大值+1 继续而非从 1 开始。

### 根因
SQLite 的 `AUTOINCREMENT` 计数器独立于数据行，`DELETE FROM` 不会重置。需显式 `DELETE FROM sqlite_sequence`。

### 修复
`app/routes/upload.py` 第219-220行，在4个 `DELETE FROM` 后增加：
```python
db.execute("DELETE FROM sqlite_sequence")
```
