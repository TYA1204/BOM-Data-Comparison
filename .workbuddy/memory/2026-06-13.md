## 叶子节点归组优化 (09:54)

**需求**：方案A — H5F中间父件不输出，仅提取H5F叶子节点合并到P3EM直接上级父组件，P3EM同样只展示叶子数据直接上级。

**实现** (`change_notice.py`):
- 新增叶子判定逻辑：PN ∈ parent_pn_set → 中间节点，否则 → 叶子
- 重写 `group_diffs_by_parent()`: 按叶子的 direct parent_pn 重新归组，不再按原始层级分组
- 移除 self-referencing（所有item均为叶子，组头已显示父件信息）
- H5F叶组通过功能键匹配合入P3EM叶组

**验证结果**: 30条diff → 14个中间节点过滤 → 16个叶子 → 5组, 0违规, 0 H5F中间父件残留, 差异=0

| 分组 | 类型 | 内容 |
|------|------|------|
| 整机非公共组件 100P5FP | P3EM+H5F合并 | 遥控器 ADD/DEL |
| 面壳组件 100P3E MAX | P3EM+H5F合并 | 型材结构件 ADD/DEL |
| 包装组件 100P3E MAX | P3EM+H5F合并 | 5项ADD+5项DEL |
| LED电源组件 | MOD | 贴片电阻 qty变更 |
| 液晶组件 100H5F PRO | P3EM | FFC ADD |

**被消除的中间分组**: 大配管BOM、电子BOM、结构BOM、液晶模组（全部为中间层级节点链）
