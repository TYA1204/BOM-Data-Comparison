## compare 界面全选/单选框联动 Bug 修复 (10:08)

**触发**：用户报告：全选取消勾选后，点击单个组件复选框错误触发全选（所有组件被选中）。

**根因分析（3 个缺陷）**：

| # | 缺陷 | 严重度 | 说明 |
|---|------|--------|------|
| 1 | **级联循环** | 🔴 严重 | `onCompCheckChange` 直接设置 `comp-toggle.checked = allChecked`。若 `allChecked=true`（如单根节点树），且浏览器在程序化赋值时派发 `change` 事件，则触发 `toggleAllComponents` → 所有复选框被勾选 |
| 2 | **状态丢失** | 🔴 严重 | `renderComponentTree` 硬编码 `checked` 属性。展开/折叠/搜索/过滤触发树重绘时，用户手动取消的勾选全部被重置为勾选状态 |
| 3 | **隐藏行遗漏** | 🟡 中等 | `toggleAllComponents` 只操作 `display !== 'none'` 的可见行。折叠节点的子项复选框不受全选/取消全选影响，展开后状态不一致 |

**修复方案（compare.html）**：
1. 新增 `_compToggleGuard` 守卫标志 → 阻断 `onCompCheckChange → toggle-all update → toggleAllComponents` 级联链
2. 新增 `compCheckedState = {source: {}, target: {}}` 缓存层 → `renderComponentTree` 重绘时读取缓存恢复状态
3. 新增 `_syncCompCheckState(side)` → 将 DOM 中所有 checkbox 状态同步到缓存
4. `loadComponents` 初始化时预填充缓存（所有有子件的节点默认 true）
5. `renderComponentTree` 渲染时用 `(pn in cache) ? cache[pn] : true` 替代硬编码 `checked`
6. `toggleAllComponents` 移除可见性过滤 → 操作全部 checkbox 并同步缓存
