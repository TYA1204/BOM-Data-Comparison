# 2026-07-16

## compare 界面新增「仅看差异组件」开关

**需求**：组件树默认展开骨架，用户希望只显示"叶子数据存在差异的父组件"及其链路，聚焦差异。

**设计决策（用户确认，未擅自扩展）**：
- 交互形态：组件面板加「仅看差异」开关按钮（默认仍全展开骨架，点一下切差异视图）
- 过滤范围：**只看叶子差异**——组件级 added/removed（diff_category='component'）不纳入 keepSet
- 联动勾选：纯展示，**不影响** selected_components / 比对范围

**实现**（纯前端，后端零改动）：
- `app/templates/compare.html`：
  - 新增全局 `diffKeepSet{source,target}` / `onlyDiffMode` / `compParentMap`
  - `buildParentMap(side)` 在 loadComponents 成功后构建 childPN→parentPN 映射
  - `buildDiffKeepSet()` 在 loadResult 拿到 allResults 后，按叶子差异收集保留集（差异叶子自身 PN + 父组件 PN + 全部祖先 PN）
  - `renderComponentTree` 新增 onlyDiff 参数：keepSet 内节点强制展开、非 keepSet 节点隐藏、全空显示"该侧无差异组件"
  - `rebuildTreeView` 传 onlyDiff；新增 `toggleOnlyDiff(side)`
  - 两个面板 toolbar 加 checkbox 开关；`app/static/css/app.css` 加 `.comp-onlydiff-toggle` 样式

**关键 Bug（已修复）**：buildDiffKeepSet 初版误用 `r.part_number`，但比对结果字段实为 `part_number_a`/`part_number_b`（无单一 part_number）。已改为按两侧分别取：added→part_number_b 入 target，removed→part_number_a 入 source，modified 两侧同号均入。

**验证**：
- JS 语法检查通过
- 自测 5 场景全过（数量变更/叶子新增/无差异/组件级增删不纳入/来源侧删除）
- 真实数据 task 43（344条）：25 组件级排除、319 叶子级纳入，两侧 keepSet 186/211，聚焦明显
- 部署：stop→清缓存→start，health_check 退出码 0，页面开关存在，bom-tree API 200
