# WORD 更改通知单导出 — 保姆级逻辑文档

> 从「点导出按钮」到「拿到 .docx 文件」，每一步干了啥，这里全讲清楚。

---

## 一、整体流程（一张图看懂）

```
点「导出 Word 更改通知单」
        │
        ▼
前端校验导出配置（阶段/订单号/数量/拟制/审核）─ 缺一个就拦
        │
        ▼

        │
        ▼
后端 generate_change_notice() 函数上场
        │
   ┌────┼────┐
   ▼    ▼    ▼

        │
        ▼
  保存 .docx → 前端下载
```

## 二、前端干了啥

### 2.1 导出配置卡片

比对完成后，侧边栏自动显示「导出配置」卡片，内嵌 5 个输入框：

```
┌─ 导出配置 ───────────── ↺ 默认 ─┐
│ 阶段 [DVT ▾]   数量 [        ]    │
│ 订单号 [                    ]     │
│ 拟制 [           ]  审核 [      ]  │
└──────────────────────────────────┘
```

- 全部留白，填一次后 `localStorage` 自动记住，下次打开自动回填
- 「↺ 默认」按钮一键清空恢复
- 不比对时不显示，不占地方

### 2.2 必填校验

点导出时，先检查卡片里 4 个必填字段：

| 字段 | 缺了会怎样 |
|------|-----------|
| 订单号 | 红色 Toast：「请先填写：订单号」，自动滚到卡片，拒绝导出 |
| 数量 | 同上 |
| 拟制人 | 同上 |
| 审核人 | 同上 |
| 阶段 | 下拉框（请选择/DVT/PVT/MP），必选 |

全部通过 → 发请求到后端。

## 三、后端干了啥（四层处理）

### 第 0 层：准备

从数据库查出：

- 比对任务信息（哪两个 BOM 比的）
- 两个 BOM 的全名和完整编码（`bom_name` + `bom_number`）

从 `bom_number`（如 `P1C100H5FP8R713002`）用正则提取：

| 提取内容 | 正则 | 示例值 |
|---------|------|--------|
| 机芯号 | `\d+[A-Z]+\d+`（锚定后缀 3 位） | `8R713` |
| 机型名 | `P1C` 后面的前缀 | `100H5FP` |

---

### 第 1 层：取数 — `get_diff_rows()`

从 `comparison_result` 表取出该任务的**全部差异行**。

按顺序排列：**先 ADD，再 DEL，最后 MOD**。同类里按原始 BOM 行号排。

同时查出每条差异的**父组件名称**，做跨 BOM 隔离：

- ADD 项的父组件名 → 去**目标 BOM** 里查（因为 ADD 是目标 BOM 新增的）
- DEL 项的父组件名 → 去**源 BOM** 里查（因为 DEL 是源 BOM 独有的）

**每条差异最终变成这样一个结构：**

```
{
  pn: "N030103-026147-001",        // 物料编码
  name: "电子配屏组件",             // 物料名称（已清洗）
  parent_pn: "N030103-025812-002", // 父组件编码
  parent_name: "大配管BOM",         // 父组件名称
  type: "added",                   // added / removed / modified
  type_label: "ADD",               // ADD / DEL / MOD
  qty: 1                           // 数量
}
```

---

### 第 2 层：洗名 — `clean_material_name()`

物料名称太长，分四道规则洗掉无用部分：

#### 第一道：精确噪声词（直接剔除）

| 类别 | 例子 |
|------|------|
| 品牌 | `SKYWORTH` `coocaa` `HKC` `WANZHONG` |
| 认证 | `RoHS` `REACH` `CBU` |
| 地区/状态 | `内销` `通用` `中国` `无` |
| 工艺类型 | `研发机贴` `研发手插` `研发机插` |
| 技术描述 | `底收声` `PDM` |
| 平台码 | `8R713` `8R710` `7T871` `7T611` |
| 紧固件描述 | `镀黑锌` `镀锌` `十字` `加硬` `非耐落` `无垫圈` `机械牙` `平尾` 等 30+ |

#### 第二道：模式噪声（正则匹配，直接剔除）

| 匹配模式 | 例子 |
|---------|------|
| `\d+mm` / `\d+\.\d+mm` | `60mm` `66.5mm` |
| `[-~]?\d+~?\d*℃?` | `125°C` `-30~105℃` |
| `N\d{6}-\d{6}-\d{3}` | `N011901-03017252` |
| `[A-Z]{2,}\d+[A-Z0-9-]*` | `CV850U1-L01`（面板型号） |

#### 第三道：复合噪声（拆解后全噪声才剔除）

如 `机械牙平尾` → 拆成 `机械牙` + `平尾`，都是噪声 → 剔除。
如 `机牙螺钉B` → 拆掉 `机牙` 后还剩 `螺钉B`（非噪声）→ 保留。

#### 第四道：截断标记（遇到就停，后面全丢）

| 标记 | 例子 |
|------|------|
| `±X%` | `±1%` `±10%` — 公差 |
| `X/YW` | `1/16W` `1/8W` — 功率等级 |
| `XW` | `230W` — 功率 |
| `XmA` | `650mA` — 电流 |
| `XCH` | `2CH` — 通道数 |
| `X7R` `Y5V` `NP0` | 介质材料 |
| `AC:xxx` | 交流规格 |
| `2V4000` 类 | 电压型号 |
| `2PC` `1PC` | 数量标记 |

#### 效果示例

```
贴片高容陶瓷电容 50V 1μF ±10% X7R 125°C
  → 贴片高容陶瓷电容 50V 1μF

机牙螺钉B 镀黑锌 十字 加硬 非耐落 无垫圈 机械牙平尾
  → 机牙螺钉B

LED二合一电源组件 L9M34E 0 通用 220V 430W 内销 3 RoHS
  → LED二合一电源组件 L9M34E
```

---

### 第 3 层：归组 — `group_diffs_by_parent()`（核心）

#### Step 1 — 圈定"爹名单"

遍历全部差异行，把所有 `parent_pn` 收集起来，形成一个集合。

> 这个集合里的 PN = "在别处当过爹"。

#### Step 2 — 分拣：叶子留下，中间跳过

逐个检查每条差异的 PN：

```
PN 在爹名单里 → 它是中间组件 → 跳过！
PN 不在爹名单里 → 它是叶子 → 按它的 parent_pn 分到对应父组件组里
```

**举个例子：**

```
差异行 A：PN=电阻100Ω，  parent_pn=主板组件X
差异行 B：PN=主板组件X，  parent_pn=电子BOM
```

- 主板组件X 在爹名单里（它是电阻100Ω的 parent_pn）→ 跳过
- 电阻100Ω 不在爹名单里 → 归入「主板组件X」组

跳过中间组件的原因是：它们只是结构挡板，业务上不关心"主板组件X 新增/删除"，只关心叶子物料的变化。

#### Step 3 — 判国籍

每个父组件组根据里面差异的类型，分三类：

| 组里只有 ADD | → P3EM 组 | 目标 BOM 独有的 |
| 组里只有 DEL | → H5F 组 | 源 BOM 独有的 |
| 混合 / 有 MOD | → MOD 组 | 两边都有的 |

#### Step 4-5 — 招安：H5F → P3EM 合并

H5F 组里的父组件不输出（它的名称只在 H5F 侧有意义），但它的叶子数据不能丢。

**怎么塞到对的地方？** 用"功能键"匹配。

取父组件名的前几个词作为功能键：

```
"电子BOM 100H5FP"  → 功能键 = "电子BOM"
"包装组件 100H5F"  → 功能键 = "包装组件"
```

然后在 P3EM 组里找功能键相同的：

```
P3EM 有 "电子BOM 100P3EM" → 功能键 = "电子BOM" → 匹配！
→ H5F 的叶子（DEL 项）全塞进 P3EM 的电子BOM 组
```

H5F 父组件名不输出，但它的叶子差异并入了功能相同的 P3EM 父组件下。匹配不上的兜底排最后。

#### Step 6 — 排序

每个组取组内所有差异的原始 BOM 行号（`line_no`）最小值，按这个排序输出。保证顺序跟 BOM 文件里的物料排列一致。

---

### 第 4 层：填表 — `_build_content_body()`

在 WORD 模板的「更改内容」大合并单元格里，逐段落往里写。

**父组件头行：**

用 `_get_functional_key()` 截取核心组件名，去掉尾部技术参数：

```
LED二合一电源组件 L8M30P 研发机贴 AC:198V-242V 230W ...
  → 显示为: 在N030102-xxx LED二合一电源组件里

麦克风组件 研发机贴 2V4000 2PC 2mic+RLED+KEY+IR ...
  → 显示为: 在N030103-xxx 麦克风组件里
```

**输出格式：**

```
在 [父组件PN] [父组件名·已截短]里
ADD: [物料PN] [物料名称]·······[数量]PC    ← 用量制表位对齐
     [物料PN] [物料名称]·······[数量]PC
DEL: [物料PN] [物料名称]·······[数量]PC

（空一行，隔开下一组）
```

**样式：**

- 父组件头行：蓝色粗体（`#1E40AF`），字号 11pt
- 物料行：深灰色（`#334155`），字号 10pt
- 字体：宋体优先，自动回退到系统可用中文字体

**用量对齐：**

每条物料行设右对齐制表位在 11cm 处，用量前插 `\t`。同一组件下不管物料名多长，`1PC`、`2PC` 自动对齐到同一竖线。

**ADD 和 DEL 的首行有标签：**

```
ADD:N030103-026147-001 电子配屏组件 85V68HP           1PC
    N030103-025054-001 机械组件                         1PC
DEL:N030103-026813-001 电子配屏组件 85V58F             1PC
```

## 四、特殊情况

### 情况 1：两边父组件 PN 一样

如「电子BOM PN 在两边相同且都在差异里」→ MOD 组，ADD 和 DEL 排在同一个父组件头下面。不走合并。

### 情况 2：纯 ADD 或纯 DEL 但功能键匹配不上

排到最后，不丢数据。

### 情况 3：没有任何叶子差异

MOD 组如果除了头没有叶子 → 不输出这个组。

### 情况 4：`P1C*` 开头的父组件

顶层装配件（如 `P1C100H5FP8R713002`）→ 不输出，直接跳。

## 五、最后一步：命名 + 保存

文件名格式：`{订单号}_{机芯}_{机型}_{数量}_{阶段}_整机更改通知单.docx`

如：`2606002KL_8R713_100P3EM_2_DVT_整机更改通知单.docx`

保存到 `reports/` 目录，返回路径给前端下载。
