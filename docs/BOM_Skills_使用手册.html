二、上传 BOM 文件

1
切到「上传BOM」页签。点页面顶部的「上传BOM」标签。
2
选择文件。点「选择文件」按钮，找到你的 BOM 文件（.xls 或 .xlsx 都可以，系统自动识别格式）。
3
上传解析。点「上传解析」，等几秒，右边会显示解析出来的 BOM 树。
4
上传第二份。同样操作，把要对比的另一份 BOM 也传上去。
两份都上传完才能去比对，少一份比不了。

三、比对操作

3.1 选择比对类型

按钮用途
跨机型比对（默认）不同机型互相对比（如 100P3EM vs 100H5F）— 最常用
版本比对同一机型新旧版本对比

3.2 选择对比范围

选项比什么什么时候用
全层级对比（含物料）所有东西都出来对比，组件+物料全上想看到每个细节差异，推荐
组件对比（排除物料）只比组件，叶子物料不参与想看结构变化

3.3 选择组件

页面左边显示两份 BOM 的组件树（电子BOM、结构BOM 等大类），默认全部勾选。不想比某个组件就把它的复选框点掉（不勾的组件及其全部子件会从比对中完全移除）。

3.4 开跑

点「开始比对」按钮。比对完成后页面显示差异明细。

比对完成后左侧会自动显示「导出配置」卡片，填好：

字段说明示例

数量填数字12

拟制你的名字杨芮

填一次就记住了，下次打开自动回填。

4.2 WORD 更改通知单

点差异明细右上角蓝色「导出 Word 更改通知单」按钮，自动生成标准格式的更改通知单。

WORD 报告特性：
• 自动按父组件分组，蓝色粗体组件头 + 深灰色物料行
• 物料名称自动清洗（去品牌/认证/规格/颜色等噪声词）
• 智能分页 — 每页约 28 行，不会出现首页整页留空
• 文件名：{订单号}_{机芯}_{机型}_{数量}台_{阶段}_整机更改通知单.docx
• 说明栏自动填入正确的 BOM 编号

4.3 Excel 报告

差异明细右上角「导出 Excel 报告」按钮，下下来是一份结构化表格，方便二次加工。

二、BOM 数据清洗（bom-data-cleaner）

这个 Skill 能干什么？

核心任务：把任何格式的 BOM 文件 → 自动识别 → 清洗 → 存进数据库。

支持 SAP 展开表 Excel CSV HTML 五种格式，系统会自动判断，不需要你手动选。

场景 A：导入 SAP BOM 展开表（最常用）

1

把 SAP 文件放到项目目录下

将 SAP 导出的 BOM 展开表文件放到 D:\04_系统开发\BOM Data Comparison\uploads\ 目录。

SAP 文件必须是从 SAP 直接导出的原始 UTF-16 LE Tab 分隔文件，不要用 Excel 打开另存过！另存后会丢失编码信息，导致识别失败。
2

在 WorkBuddy 中说出你的需求

输入以下任一指令：

导入 uploads/ 目录下的 SAP BOM 文件，进行清洗
清洗这个 BOM 展开表，文件名是 BOM_HSW504B.txt
3

查看清洗结果

AI 会返回清洗统计信息，包含：

  • 总行数：BOM 中的物料总条数
  • 独立物料数：去重后的物料种类
  • 层级分布：L1 有多少、L2 有多少……
  • 重复物料：同一 PN 在 BOM 中出现多次的列表（前20条）
  • 数据预览：前10行样例
预览数据如果层级关联正确（根物料下面跟着一级子件、二级子件），说明清洗成功。

场景 B：导入普通 Excel BOM

1

准备 Excel 文件

确保你的 Excel / CSV BOM 文件中包含以下常见列名之一：

你需要的数据可接受的列名（任一即可）
物料编码P/N、料号、物料编码、Material、Part Number
物料名称Description、描述、物料名称、Part Name
数量QTY、用量、数量、Quantity
位号Ref Des、位号、参考位号、Reference
如果你的文件中有 Level 或 层级 列，系统优先使用。没有的话系统会自动通过栈算法推导。
2

拖入 WorkBuddy 或指定路径

帮我导入这个 Excel BOM，文件在 uploads/my_bom.xlsx

清洗之后：查看 Web 页面

清洗完成后，BOM 数据已存入数据库。可以通过项目 Web 页面查看：

  1. 在 WorkBuddy 中输入：启动 BOM 项目
  2. AI 会启动 Flask 服务，点击链接打开网页
  3. 在上传页面可以看到清洗后的 BOM 列表及统计

三、BOM 组件导出（bom-component-exporter）

这个 Skill 能干什么？

核心任务：把数据库中的 BOM 数据 → 按需求生成专业格式的 Excel 报告。

四种导出模式自由切换：单BOM清单 双BOM对比 按组件拆分 差异报告

模式 1：导出单 BOM 完整清单

1

确认 BOM 已经入库

在 WorkBuddy 中问：已导入的 BOM 有哪些？ AI 会列出所有 BOM 及其 ID。

2

请求导出

导出 BOM ID 为 3 的清洗数据到 Excel

或更简单地：导出 HSW-504B 的 BOM 清单

3

打开文件

生成的 Excel 在 reports/ 目录下，文件名格式：BOM清洗数据_{名称}_{时间戳}.xlsx

Excel 内容按纵向单列、层级着色排列：

层级填充色代表含义
Root淡橙顶层整机 BOM
L1淡蓝一级组件（PCB板、外壳等）
L2淡绿二级子件（芯片、连接器等）
L3白色三级子件
L4+浅灰更底层子件

模式 2：按组件导出（采购最爱）

1

说出你的需求

按组件导出 BOM ID 3，所有 unit=ST 的组件各一个 Sheet

系统自动筛选出所有 ST 物料，每个物料一个 Sheet，Sheet 中列出该组件的全部后代子件。

2

手动指定组件

如果想只导出特定组件：

按组件导出，只导出物料号为 H000000-0000-01 和 N030103-0000-01 的子件

模式 3：双 BOM 对比

1

确认两个 BOM 都已入库

把 BOM-A 和 BOM-B 都导出到一个 Excel 里，方便对比

模式 4：差异比对报告

1

先执行比对

比对 BOM ID 2 和 BOM ID 3 的差异
2

导出差异报告

把比对结果导出成 Excel 差异报告

生成的报告包含两个 Sheet：

  • 汇总 Sheet：按新增/删除/变更 × 高/中/低 严重度的 9 格矩阵

四、典型实战场景

场景：新项目 BOM 入库

目标：拿到 SAP 展开表，入库并导出物料清单

  1. 把文件放 uploads/
  2. 导入 SAP BOM 展开表
  3. 查看清洗统计，确认无误
  4. 导出完整的 BOM 清单
  5. 拿到 reports/ 下的 Excel

全程耗时 < 2 分钟

场景：BOM 变更分析

目标：对比新旧两版 BOM，找出所有变更

  1. 分别入库新旧两个 BOM 文件
  2. 比对 BOM ID X 和 Y
  3. 查看 Web 页面中的差异表格
  4. 导出差异报告到 Excel
  5. 按严重度筛选重点关注项

全程耗时 < 3 分钟

场景：采购按组件下单

目标：给采购部门每个组件的子件清单

  1. 确认 BOM 已入库
  2. 按组件导出，unit=ST
  3. 得到每个 ST 组件一个 Sheet 的 Excel
  4. 包含该组件的所有后代子件
  5. 发给采购部门

全程耗时 < 1 分钟

场景：ECN 变更追溯

目标：确认 ECN 变更后哪些物料受影响

  1. 入库 ECN 前、后两个版本 BOM
  2. 对比两个 BOM，关注位号变更
  3. 差异报告中标红的行是高优先级
  4. 导出报告存档

全程耗时 < 3 分钟

六、常见问题 FAQ

Q: BOM 上传后用量显示为 0？

A: 已修复。系统现在支持 UTF-16 LE 和 GBK 两种 SAP 编码格式，并兼容新旧两种列位布局。如仍出现请刷新页面后重新上传。

Q: 导出的 WORD 首页大片空白？

A: 已修复。现在每约 28 行自动分页，Word 不会再把整个合并单元格内容推到下一页。如仍有问题请 Ctrl+F5 强制刷新后重新导出。

Q: WORD 中机型显示不对（如显示 3000 而非 F27G56M）？

A: 已修复。现在支持 P1C 和 PCC 两种 BOM 编码格式的机型提取。重新上传 BOM 再导出即可。

Q: 物料名称太长，不方便查阅？

A: 系统已内置名称清洗功能，自动去除品牌、认证、规格参数、颜色、合规标记等噪声词。另有限制 5 个核心词的兜底策略。

Q: 删除 BOM 时报错？

A: 已修复。删除 BOM 时会自动级联清理关联的比对任务和比对结果，不再报外键约束错误。

Q: BOM 列表数据显示不全？

A: 表格列宽已优化，12 列全部居中显示。如果屏幕较小可横向滚动查看全部列。

Q: 页面打开是旧版本？

A: 按 Ctrl + F5 强制刷新，清除浏览器缓存后再试。
