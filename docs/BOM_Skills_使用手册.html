二、BOM 数据清洗（bom-data-cleaner）

这个 Skill 能干什么？

核心任务：把任何格式的 BOM 文件 → 自动识别 → 清洗 → 存进数据库。

支持 SAP 展开表 Excel CSV HTML 五种格式，系统会自动判断，不需要你手动选。

场景 A：导入 SAP BOM 展开表（最常用）

1

把 SAP 文件放到项目目录下

将 SAP 导出的 BOM 展开表文件放到 D:\04_系统开发\BOM Data Comparison\uploads\ 目录。

SAP 文件必须是从 SAP 直接导出的原始 UTF-16 LE Tab 分隔文件，不要用 Excel 打开另存过！另存后会丢失编码信息，导致识别失败。
2

在 WorkBuddy 中说出你的需求

输入以下任一指令：

导入 uploads/ 目录下的 SAP BOM 文件，进行清洗
清洗这个 BOM 展开表，文件名是 BOM_HSW504B.txt
3

查看清洗结果

AI 会返回清洗统计信息，包含：

  • 总行数：BOM 中的物料总条数
  • 独立物料数：去重后的物料种类
  • 层级分布：L1 有多少、L2 有多少……
  • 重复物料：同一 PN 在 BOM 中出现多次的列表（前20条）
  • 数据预览：前10行样例
预览数据如果层级关联正确（根物料下面跟着一级子件、二级子件），说明清洗成功。

场景 B：导入普通 Excel BOM

1

准备 Excel 文件

确保你的 Excel / CSV BOM 文件中包含以下常见列名之一：

你需要的数据可接受的列名（任一即可）
物料编码P/N、料号、物料编码、Material、Part Number
物料名称Description、描述、物料名称、Part Name
数量QTY、用量、数量、Quantity
位号Ref Des、位号、参考位号、Reference
如果你的文件中有 Level 或 层级 列，系统优先使用。没有的话系统会自动通过栈算法推导。
2

拖入 WorkBuddy 或指定路径

帮我导入这个 Excel BOM，文件在 uploads/my_bom.xlsx

清洗之后：查看 Web 页面

清洗完成后，BOM 数据已存入数据库。可以通过项目 Web 页面查看：

  1. 在 WorkBuddy 中输入：启动 BOM 项目
  2. AI 会启动 Flask 服务，点击链接打开网页
  3. 在上传页面可以看到清洗后的 BOM 列表及统计

三、BOM 组件导出（bom-component-exporter）

这个 Skill 能干什么？

核心任务：把数据库中的 BOM 数据 → 按需求生成专业格式的 Excel 报告。

四种导出模式自由切换：单BOM清单 双BOM对比 按组件拆分 差异报告

模式 1：导出单 BOM 完整清单

1

确认 BOM 已经入库

在 WorkBuddy 中问：已导入的 BOM 有哪些？ AI 会列出所有 BOM 及其 ID。

2

请求导出

导出 BOM ID 为 3 的清洗数据到 Excel

或更简单地：导出 HSW-504B 的 BOM 清单

3

打开文件

生成的 Excel 在 reports/ 目录下，文件名格式：BOM清洗数据_{名称}_{时间戳}.xlsx

Excel 内容按纵向单列、层级着色排列：

层级填充色代表含义
Root淡橙顶层整机 BOM
L1淡蓝一级组件（PCB板、外壳等）
L2淡绿二级子件（芯片、连接器等）
L3白色三级子件
L4+浅灰更底层子件

模式 2：按组件导出（采购最爱）

1

说出你的需求

按组件导出 BOM ID 3，所有 unit=ST 的组件各一个 Sheet

系统自动筛选出所有 ST 物料，每个物料一个 Sheet，Sheet 中列出该组件的全部后代子件。

2

手动指定组件

如果想只导出特定组件：

按组件导出，只导出物料号为 H000000-0000-01 和 N030103-0000-01 的子件

模式 3：双 BOM 对比

1

确认两个 BOM 都已入库

把 BOM-A 和 BOM-B 都导出到一个 Excel 里，方便对比

模式 4：差异比对报告

1

先执行比对

比对 BOM ID 2 和 BOM ID 3 的差异
2

导出差异报告

把比对结果导出成 Excel 差异报告

生成的报告包含两个 Sheet：

  • 汇总 Sheet：按新增/删除/变更 × 高/中/低 严重度的 9 格矩阵
  • 明细 Sheet：每条差异的 16 列详细信息（物料号、变更字段、原值、新值、匹配度等）

四、典型实战场景

场景：新项目 BOM 入库

目标：拿到 SAP 展开表，入库并导出物料清单

  1. 把文件放 uploads/
  2. 导入 SAP BOM 展开表
  3. 查看清洗统计，确认无误
  4. 导出完整的 BOM 清单
  5. 拿到 reports/ 下的 Excel

全程耗时 < 2 分钟

场景：BOM 变更分析

目标：对比新旧两版 BOM，找出所有变更

  1. 分别入库新旧两个 BOM 文件
  2. 比对 BOM ID X 和 Y
  3. 查看 Web 页面中的差异表格
  4. 导出差异报告到 Excel
  5. 按严重度筛选重点关注项

全程耗时 < 3 分钟

场景：采购按组件下单

目标：给采购部门每个组件的子件清单

  1. 确认 BOM 已入库
  2. 按组件导出，unit=ST
  3. 得到每个 ST 组件一个 Sheet 的 Excel
  4. 包含该组件的所有后代子件
  5. 发给采购部门

全程耗时 < 1 分钟

场景：ECN 变更追溯

目标：确认 ECN 变更后哪些物料受影响

  1. 入库 ECN 前、后两个版本 BOM
  2. 对比两个 BOM，关注位号变更
  3. 差异报告中标红的行是高优先级
  4. 导出报告存档

全程耗时 < 3 分钟

六、常见问题 FAQ

Q: SAP 文件导入失败，提示格式无法识别？

A: 最常见的原因是文件被 Excel 打开过并另存了。请确保使用的是 SAP 直接导出的原始 UTF-16 LE 编码文件。判断方法：用记事本打开文件，如果能看到 "B O M 展开表" 字样且编码显示为 UTF-16 LE，就是正确的。

Q: Excel BOM 导入后层级不对？

A: 如果你的 Excel 中没有 Level（层级）列，系统会尝试用栈自动推导。推导依据是数据的物理行顺序（越靠前的行层级越高）。请确保你的 BOM 数据按层级从高到低的顺序排列。

Q: 导出的 Excel 中 PN 编码前面还有数字？

A: 导出时会自动清理 1、 2． 这样的前置序号，但清洗入库时的原始数据不会修改。如果导出结果不对，可以用 重新清洗 BOM ID X 中的 PN 编码 指令再次处理。

Q: 按组件导出时，找不到想要的组件？

A: 自动模式下只筛选 unit=ST 的物料作为组件。如果你的组件单位不是 ST，可以手动指定：按 PN 列表导出：物料号1, 物料号2

Q: 如何查看数据库中已有哪些 BOM？

A: 输入 列出所有已导入的 BOM 或 查看 BOM 列表，AI 会返回所有 BOM 的 ID、名称、导入时间。

Q: 文件要放在哪里？

A: 所有 BOM 源文件放在 D:\04_系统开发\BOM Data Comparison\uploads\，导出的 Excel 报告在 D:\04_系统开发\BOM Data Comparison\reports\。
